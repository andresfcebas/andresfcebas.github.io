    Main Description

    Respaldo

    Cuando se termina de realizar una tarea de Moddeado y luego de verificar el cumplimiento de los objetivos de la Modding Task, es necesario realizar un respaldo (Backup) de los archivos que fueron alterados con el fin de evitar pérdida de trabajo ante posibles imprevistos.

    Manejando un sistema de control de versiones se simplifica esta tarea, ya que de forma casi automática se tiene una línea de tiempo de versiones del mod y los cambios realizados en él cada vez. Para un sistema manual de versiones, se recomienda poseer un repositorio donde almacenar los datos con las descripciones de los cambios que se realizaron, sus fechas y el responsable de cada acción.

    Steps
    Realizar una descripción del cambio que se realizó en la tarea de Moddear
    Realizar una lista de posibles problemas que se pueden presentar por ese cambio
    Recolectar la información junto con la fecha y el responsable
    Crear un respaldo del proyecto con el cambio añadido la descripción, la fecha y el responsable
    Key Considerations

    El orden y la disciplina en esta tarea es fundamental, ya que si no se realiza la acción de crear los respaldos ordenadamente se podrán obtener datos duplicados y ausencia de la documentación de los responsables y los cambios que se realizaron.
