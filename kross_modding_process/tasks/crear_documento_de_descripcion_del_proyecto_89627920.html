Task: Crear Documento de Descripción del Proyecto
Se crea la estructura de un documento en donde se reunirá la información correspondiente a la idea, conceptos, maquetas, equipo de trabajo, entre otros ítems presentes en el proyecto.
Purpose
El propósito de esta tarea es la creación de la estructura de un documento, que recopila la información clave del proyecto. Lo anterior con el fin de tener una referencia para el desarrollo y posterior mejora.

    Documento de Descripción del Proyecto

    En este documento se recopila toda la información clave para el proyecto de modding, con el fin de referenciar aspectos importantes presentes en el desarrollo del mod.

    Se debe tener en cuenta lo siguiente:

    • En esta primera etapa del documento solo se crea el esqueleto del documento haciendo mención de los ítems propuestos en esta tarea.
    • El contenido del documento se desarrollará en la primera fase del proyecto, llamada "Pre-Modding".
    • En la tarea "Actualizar el Documento de Diseño del Mod" se agregará a este documento la información generada en tareas previas.


    • Introducción al Proyecto

    Dentro de este ítem se realiza una descripción del proyecto de modding. Este ítem cuenta con lo siguiente:

    Concepto del Mod

    Aquí se realiza una descripción general del proyecto y se responde a las siguientes preguntas:

    • ¿En qué consiste mi idea?
    • ¿Qué es lo que deseo hacer?
    Características principales

    En este punto se establecen las características que tendrá el producto y se responde a lo siguiente:

    • ¿Qué hace diferente este mod de otros de que ya existen?
    Género al que pertenece

    Se realiza una asociación del proyecto a un género, dependiendo de la idea y del producto que se moddeará. Entre los diferentes tipos de género se encuentran: shooters, RPG, estrategia, móviles, aventura, plataformas, habilidad, entre otros tipos.

    Maquetas

    Dentro de este ítem se realizan abstracciones de diseño mediante maquetas. Algunas de estas abstracciones son la interfaz del juego, paneles, menús, etc.

    Conceptos

    Se realizan algunas descripciones de los conceptos clave que tendrá el proyecto. Dentro de estos conceptos, se encuentran: personajes, armas, unidades, artefactos del juego, simbología, entre otros.

    • Datos del Producto

    Elección del Producto

    Se describe el nombre del producto, las ventajas que posee en cuanto a la idea de modificación, su comunidad asociada y el conocimiento que se posee sobre él.

    Licencias y permisos

    Se agrega la información de la licencia que posee el producto que modificará, además las restricciones y los permisos sobre este. Así, se define dentro de este ítem, el tipo de licencia que posee el producto, lo que está permitido realizar y lo que no se debe hacer dentro del proyecto de modding, con respecto al producto elegido.

    Prestaciones del Producto

    Aquí se realiza una descripción de lo que el producto ofrece para facilitar la tarea de modding. Se describe si es amigable al modding (Mod Friendly), si posee un SDK para facilitar el proceso, la documentación y las herramientas auxiliares que se pueden adquirir.

    • El Equipo de Trabajo

    Dentro de este ítem se realiza una descripción de los miembros del equipo que trabajarán en el proyecto, los roles que desempeñarán y las responsabilidades asociadas a cada uno de ellos.

    • Cambios

    Se realiza una lista de los cambios que se realizarán en el proyecto, se clasifican de la siguiente manera:

    Primarios

    Los cambios primarios son aquellos que son totalmente necesarios, generalmente tienen que ver con la jugabilidad y están asociados a la idea principal.

    Secundarios

    Los cambios secundarios son aquellos que son opcionales, generalmente aportan aspectos estéticos y están asociados con extensiones de la idea principal.
