Main Description

Manejo de Versiones

En cualquier desarrollo es importante tener un respaldo de los archivos del proyecto, ya que cualquier imprevisto puede ocasionar grandes pérdidas en cuanto a recursos de trabajo y tiempo. Configurar un sistema de respaldo y de versiones del producto es una tarea que permitirá contrarrestar posibles pérdidas ante fallos e imprevistos. Para ello se puede hacer uso de sistemas de manejo de versiones como GIT o SUBVERSION, o en cualquier otro caso, manejo de copias de seguridad en la nube o locales.

En este paso es muy importante la disciplina y el orden, ya que el manejo de versiones y copia de seguridad lo exigen. Problemas como datos corruptos, duplicación innecesaria o desordenes de versiones son muy comunes en sistemas de versión mal manejados.

Steps
Reunión con el equipo de desarrollo
Socializar sobre los sistemas de versión y copias de seguridad
Realizar una selección del sistema de versiones
Implantar el sistema de versiones en el equipo de trabajo
Key Considerations
Sistemas como GIT y SUBVERSION son muy buenas opciones para manejar este apartado.
