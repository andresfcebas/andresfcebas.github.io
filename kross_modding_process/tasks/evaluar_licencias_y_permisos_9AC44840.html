Task: Evaluar licencias y permisos
Se realiza una evaluación de los permisos y restricciones que posee el producto conforme a su licencia.
Purpose
El propósito de esta tarea es evaluar qué se encuentra permitido realizar en cuanto a modificaciones al producto, tomando como base la licencia que este posee.

      Main Description

      Licencias, permisos y restricciones

      Cada producto software posee una licencia para el usuario final. Dentro de esta licencia se encuentran los permisos y las restricciones que tiene el usuario al momento de realizar su compra. Dentro de las restricciones, generalmente se impide que el usuario pueda modificar partes internas del software que puedan comprometer los productos de la empresa desarrolladora. Un ejemplo de lo anterior es el siguiente:

      • Se tiene un videojuego adquirido en una plataforma digital. Se permite realizar modificaciones externas como mapas y traducciones, pero se prohíbe el uso de técnicas como ingeniería inversa, ya que se podrían obtener parte del código fuente, u otro tipo de recursos que son delicados para la empresa, y que no se deben dar a conocer al público.

      Otro ejemplo de problemas con licencias se presenta cuando se usan franquicias de terceros en una modificación, por ejemplo:

      • Se desea realizar una modificación a un juego de estrategia cualquiera para simular batallas de la franquicia "Star Wars". Lo anterior sería un problema dado que la empresa dueña de Star Wars podría bloquear el desarrollo del mod porque se violan derechos de autor de su franquicia.

      Así debe tenerse en cuenta el primer principio del "Kross Modding Process": El respeto. Dentro del respeto se incluye el respeto por la propiedad intelectual ajena y las restricciones que conllevan.
